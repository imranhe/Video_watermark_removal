## 三、iOS 端支付合规策略

### 3.1 Apple 政策背景

Apple 公司的《App Store 审核指南》第 3.1.1 条规定，iOS 应用内的虚拟商品（如游戏货币、积分等）必须通过 Apple 的 In-App Purchase（IAP）机制进行交易，Apple 将收取 30% 的佣金。

微信小程序作为一种特殊形态，其支付方式受到以下因素影响：
- 微信小程序运行在微信环境中，不直接通过 App Store 分发
- Apple 对微信小程序内的虚拟商品支付存在监管要求
- 微信平台自身对小程序内的支付合规有明确规范

### 3.2 合规支付方案

本小程序在 iOS 端采用 **"实物商品/服务兑换码"模式** 进行合规支付：

#### 3.2.1 方案说明

| 要素 | 说明 |
|------|------|
| **支付渠道** | 微信支付 |
| **商品性质** | 视频处理服务的预付费凭证（服务类商品） |
| **交易形式** | 用户购买的是"视频处理服务兑换券"，而非虚拟货币 |
| **发货方式** | 支付成功后系统自动发放对应积分 |
| **合规依据** | 积分作为服务预付费凭证，不属于 Apple IAP 管辖的虚拟商品范畴 |

#### 3.2.2 交易流程

```
用户发起充值 → 选择充值档位 → 微信支付完成付款
     ↓
系统收到支付回调 → 验证支付状态 → 自动发放积分
     ↓
用户收到积分到账通知 → 积分可用于视频处理服务
```

#### 3.2.3 关键合规点

1. **商品定义**: 积分定位为"视频处理服务预付费凭证"，而非通用虚拟货币
2. **服务关联**: 积分仅可用于兑换本小程序内的视频处理服务，不可用于其他用途
3. **不可转让**: 积分不可转让给其他用户，不可提现，不可兑换现金
4. **服务交付**: 充值本质是预购视频处理服务，积分只是计量单位

### 3.3 微信平台合规

根据《微信小程序运营规范》的相关要求：

1. **虚拟支付限制**: iOS 端小程序不得提供虚拟支付功能（如直接购买虚拟货币）
2. **合规替代**: 本方案将积分定义为服务预付费凭证，通过微信支付实物商品通道完成交易
3. **内容合规**: 充值页面不展示"购买积分"字样，而是展示"购买视频处理服务"或"充值服务"

### 3.4 风险提示

| 风险类型 | 风险描述 | 应对措施 |
|----------|----------|----------|
| **政策变动** | Apple 或微信可能调整虚拟商品政策 | 持续关注政策动态，及时调整方案 |
| **审核风险** | 小程序审核可能对支付方式提出异议 | 准备充分的合规说明材料 |
| **用户投诉** | 用户对积分过期等政策产生争议 | 完善用户告知流程，保留同意记录 |

## 四、退款与消费者权益保护

### 4.1 退款规则

| 情形 | 退款政策 |
|------|----------|
| 正常充值 | 不支持退款（积分属于预付费服务凭证） |
| 系统故障导致充值失败 | 自动退回支付金额至原支付方式 |
| 重复扣费 | 核实后 7 个工作日内退回多扣金额 |
| 服务未提供（平台原因） | 退还对应积分或等值金额 |
| 积分过期 | 不退还（已在充值时明确告知有效期） |

### 4.2 用户知情权保障

我们在以下环节向用户充分告知积分规则：

1. **充值页面**: 展示积分有效期、消耗规则等重要信息
2. **支付确认**: 支付前弹窗确认，展示积分数量和有效期
3. **到账通知**: 积分到账后发送通知，注明有效期
4. **到期提醒**: 积分到期前 30 天通过小程序通知提醒
5. **协议条款**: 用户服务协议中明确积分规则

### 4.3 消费者权益保护

根据《中华人民共和国消费者权益保护法》的相关规定：

- 用户享有知情权：充值前充分了解积分规则
- 用户享有选择权：可自由选择充值档位
- 用户享有公平交易权：充值规则公平透明
- 用户享有求偿权：因平台原因造成损失可申请赔偿
