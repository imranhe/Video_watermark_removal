# 后端开发指令

## 🎯 开发目标

根据 API 接口文档，开发微信小程序后端 API 服务。

## 📋 技术栈选择

**后端框架**: Node.js + Express
**数据库**: MySQL + Redis
**认证**: JWT (jsonwebtoken)
**文件上传**: Multer + 阿里云 OSS
**支付**: 微信支付 SDK

## 🔧 开发步骤

### Step 1: 初始化项目
```bash
mkdir server
cd server
npm init -y
npm install express mysql2 redis jsonwebtoken multer axios dotenv cors helmet morgan
npm install -D nodemon jest supertest
```

### Step 2: 配置数据库连接
- 创建 config/database.js
- 创建 config/redis.js
- 配置环境变量

### Step 3: 创建数据模型
- 根据数据库设计创建模型
- 定义表结构和关联

### Step 4: 实现认证模块
- 微信登录接口
- JWT Token 生成和验证
- Token 刷新接口

### Step 5: 实现用户模块
- 获取用户信息
- 更新用户信息
- 用户统计

### Step 6: 实现任务模块
- 创建任务（视频上传）
- 查询任务列表
- 查询任务详情
- 查询任务进度
- 取消/删除任务
- 下载处理后视频

### Step 7: 实现订单模块
- 获取套餐列表
- 创建订单
- 查询订单状态
- 微信支付回调

### Step 8: 实现积分模块
- 积分余额查询
- 积分记录查询

### Step 9: 实现通知模块
- 通知列表
- 标记已读

### Step 10: 实现反馈模块
- 提交反馈
- 反馈列表

### Step 11: 实现系统模块
- 系统配置
- 处理配置

### Step 12: 编写单元测试
- 为每个接口编写测试用例
- 测试覆盖率 > 80%

### Step 13: 配置 CI/CD
- GitHub Actions
- 自动测试
- 自动部署

## ⚠️ 注意事项

1. **错误处理**: 统一错误码（1001-1009）
2. **参数验证**: 使用 Joi 或 express-validator
3. **日志记录**: 使用 Morgan 或 Winston
4. **安全防护**: CORS、Helmet、Rate Limiting
5. **文件上传**: 限制大小（100MB）、格式（mp4/mov/avi）
6. **支付回调**: 验证签名、防止重复通知
7. **积分扣除**: 原子操作、防止并发问题
8. **任务队列**: 使用 Bull 或 Agenda 处理异步任务
