# 牛马去字幕项目 - 优化清单

## 📋 一、方案调整优化

### 1. 需求理解偏差（重要）
- ❌ 不需要复刻前端 UI
- ✅ 只了解布局即可
- ✅ 重点是功能实现和后端逻辑

### 2. 视频处理 API 方案调整
- ⚠️ 阿里云 SAM API 可能不支持"去字幕"
- ✅ 推荐方案：
  - 方案A：第三方 API（快速上线）
  - 方案B：自建 FFmpeg + AI 模型（长期可控）
- ✅ 先用第三方 API 验证市场，再考虑自建

### 3. 技术栈优化
- ⚠️ 当前：Node.js + Express
- ✅ 推荐：Python + FastAPI（更适合 AI 任务）
- 或者：保留 Node.js，但视频处理部分用 Python

### 4. 成本估算修正
- ⚠️ 当前估算：~¥700/月（偏低）
- ✅ 实际可能：¥10000-20000/月（按分钟计费）
- ✅ 需要设置每日限额、积分消耗控制

### 5. 分阶段上线策略
```
Phase 1: MVP（最小可行产品）
├── 只做去字幕功能
├── 简化 UI
├── 积分制付费
└── 目标：2 周内上线

Phase 2: 功能完善
├── 增加去水印、去图标
├── 优化 UI 体验
├── 增加会员制
└── 目标：上线后 1 个月

Phase 3: 高级功能
├── 自建 AI 模型
├── 批量处理
├── API 开放
└── 目标：上线后 3 个月
```

### 需要新增的表

#### 1. packages 表（套餐配置）
```sql
CREATE TABLE IF NOT EXISTS packages (
    id VARCHAR(36) PRIMARY KEY,
    name VARCHAR(50) NOT NULL COMMENT '套餐名称',
    type ENUM('points', 'monthly', 'quarterly', 'yearly') NOT NULL COMMENT '套餐类型',
    price DECIMAL(10,2) NOT NULL COMMENT '价格',
    credits INT COMMENT '积分数量（积分套餐）',
    duration_days INT COMMENT '有效天数（会员套餐）',
    description VARCHAR(200) COMMENT '描述',
    is_active BOOLEAN DEFAULT TRUE COMMENT '是否启用',
    sort_order INT DEFAULT 0 COMMENT '排序',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='套餐配置表';
```

#### 2. processing_config 表（处理配置）
```sql
CREATE TABLE IF NOT EXISTS processing_config (
    id VARCHAR(36) PRIMARY KEY,
    task_type ENUM('subtitle', 'watermark', 'logo') NOT NULL COMMENT '处理类型',
    points_cost INT NOT NULL COMMENT '消耗积分',
    max_duration_seconds INT COMMENT '最大视频时长（秒）',
    max_file_size_mb INT COMMENT '最大文件大小（MB）',
    description VARCHAR(200) COMMENT '描述',
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='处理配置表';
```

#### 3. notifications 表（通知消息）
```sql
CREATE TABLE IF NOT EXISTS notifications (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    type ENUM('task_complete', 'task_failed', 'points_low', 'system') NOT NULL,
    title VARCHAR(100) NOT NULL,
    content TEXT,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_is_read (is_read),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='通知消息表';
```

#### 4. feedbacks 表（用户反馈）
```sql
CREATE TABLE IF NOT EXISTS feedbacks (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    type ENUM('bug', 'feature', 'complaint', 'other') NOT NULL,
    content TEXT NOT NULL,
    contact VARCHAR(100) COMMENT '联系方式',
    status ENUM('pending', 'processing', 'resolved') DEFAULT 'pending',
    reply TEXT COMMENT '回复内容',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_status (status),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='反馈表';
```

#### 5. points_logs 表（积分日志，替代 usage_logs）
```sql
CREATE TABLE IF NOT EXISTS points_logs (
    id VARCHAR(36) PRIMARY KEY,
    user_id VARCHAR(36) NOT NULL,
    task_id VARCHAR(36) COMMENT '关联任务（消费时）',
    order_id VARCHAR(36) COMMENT '关联订单（充值时）',
    change_amount INT NOT NULL COMMENT '积分变动（正数增加，负数减少）',
    balance_after INT NOT NULL COMMENT '变动后余额',
    type ENUM('consume', 'recharge', 'gift', 'refund') NOT NULL COMMENT '变动类型',
    description VARCHAR(200) COMMENT '描述',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_type (type),
    INDEX idx_created_at (created_at),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='积分日志表';
```
